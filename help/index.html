Stretch Timer — URL API

Load routines via URL for AI agents and automation

Usage

Append ?routine=<base64> to the app URL to load a routine directly into the player. The value is a base64-encoded JSON string. The routine is ephemeral — it is not saved and will be gone on the next page load. 

https://<host>/?routine=<base64-encoded-json>

JSON Schema

{
  "name": "string (required, max 50 chars)",
  "exercises": [
    {
      "name":     "string (required, max 60 chars)",
      "type":     "time" | "reps" (required),
      "duration": number (seconds, 5–600, required when type="time"),
      "reps":     number (1–200, required when type="reps"),
      "rest":     number (seconds, 0–120, optional, default 0)
    }
  ]
}

No id field needed — one is generated automatically. At least one exercise is required. Fields irrelevant to the type (e.g. duration for reps) are ignored.

Example

Routine JSON:

{
  "name": "Quick Stretch",
  "exercises": [
    { "name": "Neck Rolls",        "type": "reps", "reps": 10, "rest": 5 },
    { "name": "Hamstring Stretch",  "type": "time", "duration": 30, "rest": 10 },
    { "name": "Deep Breathing",     "type": "time", "duration": 60 }
  ]
}

Base64 encode and append to URL:

echo '{"name":"Quick Stretch","exercises":[{"name":"Neck Rolls","type":"reps","reps":10,"rest":5},{"name":"Hamstring Stretch","type":"time","duration":30,"rest":10},{"name":"Deep Breathing","type":"time","duration":60}]}' | base64

Or in JavaScript:

const url = baseURL + '?routine=' + btoa(JSON.stringify(routine));

For AI Agents

To create and launch a stretch routine:

  1. Build a routine JSON object matching the schema above
  2. Base64-encode the JSON string
  3. Open <app-url>?routine=<encoded> in the user's browser

The player starts immediately — no user interaction needed beyond the initial page load.
